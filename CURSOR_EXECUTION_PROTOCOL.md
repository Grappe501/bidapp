# Cursor execution protocol

Cursor must:

- Follow instructions exactly for the active build packet
- Not expand scope beyond the packet’s objective and acceptance criteria
- Not refactor unrelated files or stable foundations without explicit instruction
- Not introduce new dependencies without approval
- Maintain the agreed folder structure and naming conventions
- Prefer small, reviewable changes with clear boundaries between UI, routing, and services

When in doubt, ask for clarification rather than inventing product behavior or architecture.
