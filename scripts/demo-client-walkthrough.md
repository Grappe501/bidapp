# Recommended client demo flow

Suggested order for **stakeholder walkthroughs** when `VITE_DEMO_MODE=true`. Not a wizard — a narrative path through the strongest surfaces.

1. **Dashboard** (`/`) — Command overview, readiness headline, recommended architecture line, next actions.
2. **AllCare profile** (`/control/intelligence`) — How the workspace understands AllCare (capabilities, services, technology).
3. **Architecture** (`/architecture`) — Stack and vendor positioning.
4. **Drafts** (`/drafts` and a section) — Generated or saved section preview.
5. **Client review** (`/output/client-review`) — Executive brief centerpiece.
6. **Output & readiness** (`/output`) — Finalization command view; then **Final readiness** / submission package subpages as needed.

Optional: **Review** (`/review`) for issue posture; **Documents** (`/files`) if asked about solicitation materials.

Enable demo mode in the **Vite build** env (`VITE_DEMO_MODE=true`, optional `VITE_DEMO_CLIENT_NAME`), then rebuild.
